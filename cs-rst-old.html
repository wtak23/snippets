GOTCHA’s with .. contents::Â¶

  • adding the :depth: int syntax is fine
  • adding the :backlinks: {entry,top,none} screws up the links in the TOC
  • using auto-section numbering with .. sectnum:: screws up the linking of TOC

h1 h2 ... problemsÂ¶

  • In github, you need to add some text between headers h1, h2, etc - blank lines will mess up the TOC structure. - I generally insert ... just for the sake of having some text in between...
  • You cannot jump from h1 to h3 without h2 in between - Github won’t even try to render

Including styles on HEADER-NAMES will break the TOC link on github (unconfirmed)Â¶

roles in RSTÂ¶

Ref: http://docutils.sourceforge.net/docs/ref/rst/roles.html

  • Basic syntax: ROLENAME:`INTERPRETED-TEXT` (note the use of the backtick ` in the second-half)
